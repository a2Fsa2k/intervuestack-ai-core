# IntervueStack AI Core — Recent Upgrades (2026-04-27)

This document summarizes the recent end-to-end upgrades made to the AI Interviewer experience. The goal was **demo-stage, production-quality behavior** (robustness, responsiveness, less spammy interventions) while **preserving the existing UI/session flow** and avoiding heavy infra work.

---

## 1) OpenAI migration (removing Gemini)

### What changed
- Replaced all Gemini-based LLM usage with **OpenAI** calls.
- Standardized model usage to **`gpt-4o-mini`** for fast, high-quality demo responsiveness.
- Added hardened JSON handling to support agents that require structured outputs.

### Why
- Keep a single reliable provider for both **LLM** and **Whisper STT**.
- Reduce brittleness from partial/invalid JSON responses.
- Improve consistency across orchestrator/persona/evaluator/feedback.

### Where
- **Client helper:** `src/components/classroom/ai/openaiClient.ts`
  - `generateOpenAIText(...)`: standard chat-completions wrapper.
  - `generateOpenAIJSON<T>(...)`: best-effort JSON parsing with:
    - JSON object extraction from mixed text
    - optional schema validation (`validate` type guard)
    - single retry with stricter instruction
    - optional fallback on repeated failures

> Note: this is currently a **browser-key demo** via `VITE_OPENAI_API_KEY` and `dangerouslyAllowBrowser: true`. A server proxy can be added later for real production key safety.

---

## 2) Whisper Speech-to-Text (STT) integration

### What changed
- Implemented OpenAI **Whisper** STT and connected it to the existing push-to-talk UX.
- Removed the earlier STT path (previously ElevenLabs).

### Why
- Better transcription quality and simpler vendor surface area.
- More credible “AI interviewer” demo: reliable voice input is a big UX multiplier.

### Where
- STT implementation: `src/components/classroom/ai/stt/openaiStt.ts`
- UI wiring + transcript handling: `src/components/classroom/ai/TranscriptPanel.tsx`

---

## 3) Real code monitor agent (deterministic, no LLM)

### What changed
- Added a **local, deterministic** agent that monitors code changes and emits signals.
- This agent does not call the LLM (it is fast, cheap, and reliable).

### Why
- Prevent “LLM-driven spam”: the system should only interrupt when it has a reason.
- Make interventions feel grounded in observable behavior (syntax issues, thrash, idle time).

### Signals/heuristics implemented
- Syntax validity checks (lightweight parse) → `syntax_error`
- Inactivity after meaningful changes (coding/testing/optimization) → `stuck_no_progress`
- Nested loops for medium problems → `likely_bruteforce` (heuristic)
- Return statement + syntax OK + meaningful diff → `solution_progress`
- Large rewrite churn → `rapid_rewrite`

### Where
- Agent: `src/components/classroom/ai/architecture/agents/codeMonitorAgent.ts`

---

## 4) Controller integration (debounced, cooldown-based interventions)

### What changed
- Integrated `codeMonitorAgent` into the main interview controller.
- Debounced monitoring so it **never runs on every keystroke**.
- Added intervention cooldown so the assistant **does not repeatedly interrupt**.
- Tracks repeated syntax errors (streak-based) to detect persistent blocking issues.

### Why
- Improves perceived intelligence: fewer but higher-value interventions.
- Improves performance: less compute + fewer rerenders.

### Where
- Controller: `src/components/classroom/ai/useInterviewController.ts`
  - Debounce: ~900ms
  - Cooldown: ~75s
  - Code-monitor signals are forwarded via internal tags like:
    - `[CODE_MONITOR] stuck_no_progress, likely_bruteforce, ...`

---

## 5) Rolling transcript summary (long-session context compression)

### What changed
- Added a rolling summary of the transcript that updates periodically.
- Orchestrator uses a **short transcript tail** + **rolling summary** instead of full transcript.

### Why
- Prevent context bloat and slow responses in longer sessions.
- Maintain continuity and memory while keeping prompts compact.

### Where
- Stored in the main store and updated via OpenAI summarization.
- Wired through `useInterviewController.ts` and consumed by `orchestratorAgent.ts`.

---

## 6) Orchestrator upgrades (smarter, less spammy)

### What changed
- Refactored orchestrator prompts and decision logic to incorporate:
  - rolling summary
  - transcript tail
  - code-monitor signals
  - code truncation
- Made interventions more selective and context-aware.

### Why
- The orchestrator should behave like a strong interviewer:
  - don’t over-talk
  - nudge only when the candidate is blocked or heading toward a weak approach
  - stay aligned with the current interview phase

### Where
- Orchestrator: `src/components/classroom/ai/architecture/orchestratorAgent.ts`

---

## 7) Persona/evaluator/feedback generation modernized

### What changed
- Updated persona shaping, evaluation, and feedback generation to OpenAI.
- Used the same JSON safety utilities for structured outputs.

### Why
- Consistent provider + consistent robustness patterns.
- Structured outputs make UI/state management safer.

### Where
- Persona: `src/components/classroom/ai/architecture/agents/personaAgent.ts`
- Evaluator: `src/components/classroom/ai/architecture/agents/evaluatorAgent.ts`
- Feedback: `src/components/classroom/ai/feedback/feedbackGenerator.ts`

---

## 8) Problem metadata improvements (supports richer interview behavior)

### What changed
- Allowed problem objects to include richer metadata (optional) such as:
  - `approaches`
  - `commonMistakes`
  - `hintLadder`
  - `probeQuestions`

### Why
- Enables more “interviewer-like” behavior without additional model calls.
- Gives the orchestrator/evaluator higher-quality grounding.

### Where
- Problem bank: `src/components/classroom/ai/problems/dsaProblems.ts`
- Question bank agent (selects problems): `src/components/classroom/ai/architecture/agents/questionBankAgent.ts`
- Passed to code monitor as `problemMetadata` from `useInterviewController.ts`.

## Talking points for the call (how to defend the changes)

### Theme: fewer interruptions, higher signal
- A deterministic code monitor avoids calling the LLM for things a compiler/heuristic can detect.
- Cooldown + debounce prevents spam and improves perceived intelligence.

### Theme: long-session reliability
- Rolling summary + transcript tail prevents prompt bloat, keeps latency stable.

### Theme: robust structured outputs
- JSON extraction/validation + retry is critical because agents are only as reliable as their parsing.

### Theme: improved demo UX
- Whisper STT dramatically improves realism and usability for an “AI interviewer” demo.
